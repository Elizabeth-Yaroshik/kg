# Лабораторная работа №3

## 1. Введение
**Цель работы:** Закрепление теоретического материала и практическое освоение основных возможностей базовых алгоритмов растеризации отрезков и кривых.

**Задачи:**
* Реализация шести алгоритмов растеризации
* Создание графического интерфейса для визуализации работы алгоритмов
* Анализ временных характеристик алгоритмов
* Обеспечение удобного интерфейса с системой координат и отображением точек

## 2. Реализованные алгоритмы
В ходе работы были реализованы следующие алгоритмы растеризации:

* **Алгоритм ЦДА (Цифровой Дифференциальный Анализатор)** - вычисляет приращения координат и последовательно добавляет точки с округлением до ближайших целых значений. Использует вычисления с плавающей точкой.

* **Алгоритм Брезенхема для отрезков** - использует целочисленную арифметику и переменную ошибки для определения следующей точки без вычислений с плавающей точкой. Наиболее эффективный алгоритм для построения отрезков.

* **Алгоритм Брезенхема для окружности** - строит окружность по 8 точкам симметрии, используя целочисленные вычисления для определения следующей точки. Основан на решении в одном октанте с последующим отражением точек.

* **Пошаговый алгоритм** - определяет доминирующую ось (X или Y) и вычисляет координаты точек вдоль этой оси, корректируя координаты по второй оси на основе накопленной ошибки.

* **Алгоритм Кастла-Питвея** - модификация алгоритма Брезенхема с использованием целочисленной арифметики и различными формулами ошибки для горизонтальных и вертикальных доминирующих осей.

* **Алгоритм сглаживания Ву** - алгоритм с антиалиасингом, который рисует две точки с разной интенсивностью для создания плавных переходов. Использует вычисления с плавающей точкой для определения интенсивности пикселей.

## 3. Временные характеристики
Для анализа производительности были проведены замеры времени выполнения каждого алгоритма с использованием `System.nanoTime()`. Приложение автоматически измеряет и отображает время выполнения для каждого построения.

**Особенности реализации измерений:**
- Время замеряется непосредственно при выполнении алгоритма
- Результаты выводятся в текстовую область интерфейса и на панель управления
- Учитывается количество построенных точек
- Отображается время в наносекундах и миллисекундах

**Сравнительные характеристики:**
- Алгоритм Брезенхема для отрезков показывает наилучшую производительность благодаря целочисленной арифметике
- Алгоритм ЦДА работает медленнее из-за операций с плавающей точкой
- Алгоритм Кастла-Питвея демонстрирует сравнимую с Брезенхемом производительность
- Алгоритм сглаживания Ву работает медленнее из-за сложных вычислений и необходимости обработки двух точек на каждом шаге
- Алгоритм Брезенхема для окружности эффективен благодаря симметрии и целочисленным вычислениям

## 4. Графический интерфейс. Особенности реализации

### Архитектура приложения:
* **Главное окно** - разделено на две основные области с помощью `JSplitPane`
* **Графическая панель** (`RasterPanel`) - левая часть, отвечает за отображение координатной сетки и точек
* **Текстовая область** - правая часть, отображает подробную информацию о выполнении алгоритма
* **Панель управления** - верхняя часть, содержит все элементы управления

### Система координат:
* Используется масштабируемая сетка с регулируемым размером ячейки (5-60 пикселей)
* Начало координат расположено в центре панели
* Оси координат подписаны с автоматическим расчетом шага
* Подписи осей отображают логические координаты

### Элементы управления (верхняя панель):
* **Выбор алгоритма** - выпадающий список с шестью реализованными алгоритмами
* **Регулятор масштаба** - слайдер для изменения размера сетки (5-60 px/ед.)
* **Поля ввода** - координаты x1, y1, x2, y2 и радиус окружности
* **Кнопки управления** - "Нарисовать" и "Очистить"
* **Индикатор времени** - отображение времени выполнения алгоритма

### Визуализация:
* **Графическая панель** - основная область для отображения алгоритмов с двойной буферизацией
* **Точки** - отображаются красным цветом с черной обводкой, размер адаптируется к масштабу
* **Точки Ву** - отображаются с градациями красного цвета в зависимости от интенсивности
* **Координатная сетка** - отображается светло-серым цветом
* **Оси координат** - выделены черным цветом с подписями

### Дополнительные функции:
* **Динамическое масштабирование** - изменение масштаба сетки в реальном времени
* **Автоматическая очистка** - полный сброс графики и текстовой информации
* **Валидация ввода** - обработка числовых значений координат
* **Адаптивный размер точек** - точки меняют размер в зависимости от масштаба
* **Подробный вывод информации** - отображение координат точек, времени выполнения и параметров алгоритма

## 5. Особенности реализации алгоритмов

### Алгоритм ЦДА:
- Возвращает список точек в логических координатах
- Обрабатывает случай нулевой длины отрезка
- Использует вычисления с плавающей точкой
- Округляет координаты до ближайших целых значений

### Алгоритм Брезенхема для линии:
- Целочисленные вычисления
- Эффективное определение следующей точки через переменную ошибки
- Обрабатывает все октанты

### Алгоритм Брезенхема для окружности:
- Построение по 8 точкам симметрии
- Использование целочисленного критерия решения
- Эффективное использование симметрии окружности

### Пошаговый алгоритм:
- Определение главной оси (горизонтальной или вертикальной)
- Построение вдоль главной оси с коррекцией по второй оси
- Использование вычислений с плавающей точкой

### Алгоритм Кастла-Питвея:
- Модификация алгоритма Брезенхема
- Различные формулы ошибки для разных случаев
- Целочисленные вычисления

### Алгоритм сглаживания Ву:
- Реализация антиалиасинга
- Рисование двух точек с разной интенсивностью на каждом шаге
- Обработка steep и shallow линий
- Использование дробной части для определения интенсивности

## 7. Заключение
В ходе работы были успешно реализованы и протестированы шесть алгоритмов растеризации. Созданное приложение позволяет:

* Наглядно изучать принципы работы различных алгоритмов растеризации
* Сравнивать производительность алгоритмов через измерение времени выполнения
* Анализировать получаемые результаты через детальное отображение координат точек и их интенсивности
